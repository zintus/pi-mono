# packages/ai

Unified LLM provider abstraction. User-facing docs live in `README.md`. Internal architecture and file map live in `src/AGENTS.md` and `src/providers/AGENTS.md` — read those first for anything code-level.

## Fast lookups

- How the streaming API is composed → `src/AGENTS.md`
- Adding a new provider → root `AGENTS.md` § "Adding a New LLM Provider" (7-step checklist)
- Per-provider quirks (OAuth stealth mode, compat layer, adaptive thinking, etc.) → `src/providers/AGENTS.md`
- Env vars, auth, browser caveats → `README.md` § "Browser Usage" / "OAuth Providers"

## Running tests

From this directory only:
```bash
npx tsx ../../node_modules/vitest/dist/cli.js --run test/<file>.test.ts
```
Root `npm run check` skips tests. `npm test` is forbidden by fork rules. Tests hitting real APIs are gated on env vars; most live-network tests in `test/` will skip without credentials.

## Model catalog

`src/models.generated.ts` (~360 KB) is regenerated by `npm run build` via `scripts/generate-models.ts`. Don't edit manually. After `~/spaceship/update-pi.sh` or any upstream rebase, the file legitimately churns — unrelated deltas there aren't your work. OpenRouter entries fall back to `max_completion_tokens || 4096` when the provider reports `null`, which can silently cap output tokens; check `scripts/generate-models.ts` before blaming a model regression.

## Fork-specific

- `PI_BEDROCK_DUMP_DIR=<dir>` — dumps every Bedrock `ConverseStream` request + stream events + terminal state as JSONL (`src/providers/amazon-bedrock.ts`). Use to diagnose suspected Bedrock-side hangs.
- `bedrock-provider.{js,d.ts}` at the package root are thin `export * from "./dist/bedrock-provider.js"` stubs, not source; the package subpath export `./bedrock-provider` resolves here.
